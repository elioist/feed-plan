## Why

实测发现移动端点单链路不可用或失败原因不可见，影响点菜服务的核心闭环。同时，当前用户与角色权限管理过于固定：虽然已有基础用户列表和角色修改，但角色、权限点和用户授权都写死在代码或页面里，无法按实际使用场景扩展；也缺少密码修改、重置密码和更清晰的用户维护流程。

本次变更把点单可用性、账号维护能力和动态 RBAC 一起补齐。角色、权限点、角色-权限授权、用户-角色授权都必须由数据库和后台页面配置，不再固定为 `chef` / `diner` 两个角色，也不在 shared 或端侧页面维护固定权限矩阵。

## What Changes

- 修复并增强移动端点单链路：明确 API base 配置、网络/认证/业务错误提示、购物车提交后的状态刷新和失败可诊断性。
- 完善用户管理：有用户管理权限的用户可新增用户、维护用户多角色授权、删除符合条件的用户，并获得明确错误反馈。
- 增加密码管理：登录用户可修改自己的密码；有用户管理权限的用户可重置其他用户密码；禁止危险自操作。
- 改善角色管理：后台支持角色搜索、新增、编辑、删除、分配权限点；角色来自数据库，不再固定为 `chef` / `diner`。
- 增加权限点管理：后台支持权限点搜索、新增、编辑、删除；权限点来自数据库，不在 shared 中写死。
- 后端鉴权改为权限驱动：JWT 携带用户角色摘要和权限点摘要；保护接口按用户实际拥有权限校验，不再使用固定角色判断。
- 同步更新 `packages/shared` DTO/schema、`packages/api-client` resource、后端接口、admin 页面和 mobile 错误处理。

## Capabilities

### New Capabilities

- None

### Modified Capabilities

- `meal-ordering`: 点单链路必须在移动端可配置后端地址、暴露失败原因，并保持购物车/菜单状态一致。
- `user-auth`: 用户管理扩展为支持新增用户、多角色授权、当前用户改密码、重置用户密码、动态角色和动态权限点。
- `admin-web`: 后台用户、角色和权限页面必须支持新增、搜索、授权、密码重置和清晰错误反馈。
- `mobile-meal-ordering`: 移动端点单失败时必须给出可操作反馈，并在网络/API 配置异常时帮助定位问题。

## Impact

- `packages/shared`: 新增或调整用户、密码、角色、权限点、授权相关 schema/type；移除固定角色/权限配置。
- `packages/db`: 新增角色、权限点、用户角色、角色权限、权限动作映射表及迁移。
- `packages/api-client`: 扩展 users/auth/roles/permissions resource，补齐改密码、重置密码和授权接口封装。
- `apps/server`: 扩展用户、角色、权限点与密码管理接口，补充权限驱动 guard、密码哈希、测试。
- `apps/admin-web`: 完善用户管理页、角色页、query/mutation 和测试。
- `apps/mobile`: 修复点单错误处理、API base 配置提示、请求失败反馈和相关状态刷新。
- `README` 或端侧配置说明：补充 Expo 真机/模拟器访问后端的配置方式。
