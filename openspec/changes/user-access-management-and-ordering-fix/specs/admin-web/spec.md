## MODIFIED Requirements

### Requirement: 用户与系统管理页面

系统 SHALL 提供用户管理、角色管理、权限点管理、系统设置和主题设置入口。仅具备对应管理权限的用户可以访问管理能力。用户管理 MUST 支持查看用户列表、搜索用户、新增用户、维护用户多角色授权、删除符合条件的用户、重置其他用户密码；角色管理 MUST 支持搜索、新增、编辑、删除角色和勾选权限点；权限点管理 MUST 支持搜索、新增、编辑、删除权限点；系统设置 MUST 至少包含主题切换。

#### Scenario: 查看用户管理
- **WHEN** 具备用户管理权限的用户进入用户管理页面
- **THEN** 系统展示用户列表、角色信息、搜索筛选项和可用操作

#### Scenario: 新增用户
- **WHEN** 授权用户在用户管理页面提交合法用户名、初始密码和多个角色
- **THEN** 系统创建用户，刷新用户列表，并展示成功反馈

#### Scenario: 授权用户多角色
- **WHEN** 授权用户在用户管理页面修改其他用户角色集合
- **THEN** 系统调用用户角色更新 API，刷新用户列表，并展示成功反馈

#### Scenario: 禁止移除自己的最后管理权限
- **WHEN** 授权用户查看自己的用户行
- **THEN** 系统禁用或拦截会导致自己失去管理权限的角色修改、删除和重置密码入口，并展示本人标识

#### Scenario: 重置用户密码
- **WHEN** 授权用户为其他用户提交合法新密码
- **THEN** 系统调用重置密码 API，关闭弹窗，并展示成功反馈

#### Scenario: 用户操作失败
- **WHEN** 创建用户、修改角色、删除用户或重置密码 API 返回校验、权限或冲突错误
- **THEN** 系统展示来自 API 的可读错误信息，并保持列表可刷新

#### Scenario: 无权限不能管理用户
- **WHEN** 不具备用户管理权限的用户访问用户管理页面
- **THEN** 系统拒绝访问或隐藏管理入口

#### Scenario: 查看角色管理
- **WHEN** 具备角色管理权限的用户进入角色管理页面
- **THEN** 系统从 API 获取角色和权限点，展示搜索、新增、编辑、删除和权限勾选能力

#### Scenario: 新增或编辑角色
- **WHEN** 授权用户提交合法角色 key、名称和描述
- **THEN** 系统保存角色配置，刷新角色列表，并展示成功反馈

#### Scenario: 角色权限点授权
- **WHEN** 授权用户为角色勾选权限点并保存
- **THEN** 系统调用角色权限更新 API，并刷新角色详情

#### Scenario: 查看权限点管理
- **WHEN** 具备权限点管理权限的用户进入权限点管理区域
- **THEN** 系统展示权限点列表、搜索、新增、编辑和删除入口

#### Scenario: 权限点不写死
- **WHEN** 后端新增、修改或删除权限点
- **THEN** 角色管理页面从 API 同步变化，无需在页面或 shared 常量中重复维护文案
