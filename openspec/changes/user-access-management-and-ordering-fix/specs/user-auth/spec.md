## ADDED Requirements

### Requirement: 用户管理

系统 SHALL 提供仅具备用户管理权限的用户可访问的用户管理能力。授权用户 MUST 能够创建用户、查看用户列表、搜索用户、为用户分配多个角色、删除未被业务数据限制的用户。系统 MUST 禁止用户移除自己的最后一个有效管理权限，避免锁死管理入口。

#### Scenario: 创建用户并分配多角色
- **WHEN** 具备用户管理权限的用户提交合法用户名、初始密码和角色 ID 列表
- **THEN** 系统创建用户，密码以 bcrypt 哈希存储，绑定提交的多个角色，响应体不包含密码哈希

#### Scenario: 用户名重复
- **WHEN** 授权用户创建已存在用户名的用户
- **THEN** 系统返回 409，且不创建重复用户

#### Scenario: 搜索用户
- **WHEN** 授权用户按用户名关键字或角色筛选请求用户列表
- **THEN** 系统返回匹配条件的用户列表，并包含每个用户当前绑定的角色摘要

#### Scenario: 修改用户角色
- **WHEN** 授权用户将其他用户的角色集合更新为多个角色
- **THEN** 系统保存用户-角色关系，并在后续登录或获取当前用户信息时返回新的角色和权限摘要

#### Scenario: 禁止移除自己的最后管理权限
- **WHEN** 授权用户尝试通过修改自己的角色集合导致自己失去用户/角色/权限管理能力
- **THEN** 系统返回 409，且当前用户授权保持不变

#### Scenario: 无权限访问用户管理
- **WHEN** 不具备用户管理权限的用户请求用户管理接口
- **THEN** 系统返回 403

### Requirement: 密码管理

系统 SHALL 支持登录用户修改自己的密码，并支持具备用户管理权限的用户重置其他用户密码。所有新密码 MUST 使用共享 schema 校验并以 bcrypt 哈希保存。系统 MUST NOT 在任何响应中返回明文密码或密码哈希。

#### Scenario: 当前用户修改密码
- **WHEN** 已登录用户提交正确旧密码和合法新密码
- **THEN** 系统更新当前用户密码哈希，并返回成功结果

#### Scenario: 旧密码错误
- **WHEN** 已登录用户提交错误旧密码修改密码
- **THEN** 系统返回 401 或 400，且密码保持不变

#### Scenario: 重置他人密码
- **WHEN** 具备用户管理权限的用户为其他用户提交合法新密码
- **THEN** 系统更新目标用户密码哈希，并返回成功结果

#### Scenario: 禁止通过管理接口重置自己的密码
- **WHEN** 用户通过用户管理接口重置自己的密码
- **THEN** 系统返回 409，并提示应使用修改自己密码流程

#### Scenario: 无权限重置他人密码
- **WHEN** 不具备用户管理权限的用户请求重置其他用户密码
- **THEN** 系统返回 403

### Requirement: 动态角色管理

系统 SHALL 将角色作为数据库配置管理。角色 MUST 支持搜索、新增、编辑、删除和权限点授权。系统 MUST NOT 将可用角色固定为 `chef` / `diner`。

#### Scenario: 创建角色
- **WHEN** 具备角色管理权限的用户提交合法角色 key、名称和描述
- **THEN** 系统创建角色，并可在用户授权和角色列表中选择

#### Scenario: 角色 key 重复
- **WHEN** 授权用户创建或修改为已存在的角色 key
- **THEN** 系统返回 409

#### Scenario: 为角色授权权限点
- **WHEN** 授权用户为角色提交权限点 ID 列表
- **THEN** 系统保存角色-权限点关系，并影响后续用户权限摘要

#### Scenario: 删除被用户使用的角色
- **WHEN** 授权用户删除仍被用户绑定的角色
- **THEN** 系统返回 409，且不删除角色

#### Scenario: 禁止删除最后管理角色
- **WHEN** 授权用户删除角色会导致系统没有任何用户具备用户/角色/权限管理能力
- **THEN** 系统返回 409

### Requirement: 动态权限点管理

系统 SHALL 将权限点作为数据库配置管理。权限点 MUST 支持搜索、新增、编辑、删除。系统 MUST NOT 在 shared、admin 页面或 mobile 页面中维护固定权限点矩阵。

#### Scenario: 创建权限点
- **WHEN** 具备权限点管理权限的用户提交合法权限 key、名称和描述
- **THEN** 系统创建权限点，并可在角色授权中选择

#### Scenario: 权限点 key 重复
- **WHEN** 授权用户创建或修改为已存在的权限 key
- **THEN** 系统返回 409

#### Scenario: 删除被角色使用的权限点
- **WHEN** 授权用户删除仍被角色绑定的权限点
- **THEN** 系统返回 409，且不删除权限点

#### Scenario: 无权限管理权限点
- **WHEN** 不具备权限点管理权限的用户请求权限点管理接口
- **THEN** 系统返回 403
