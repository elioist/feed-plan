## MODIFIED Requirements

### Requirement: 用户与系统管理页面

系统 SHALL 提供用户管理、角色管理、系统设置和主题设置入口。仅 chef 可以访问管理能力。用户管理 MUST 支持查看用户列表、新增用户、编辑用户（头像、用户名）、维护用户角色、重置密码和删除用户；角色管理 MUST 基于共享角色配置展示系统支持的 chef 与 diner 角色及其能力边界；系统设置 MUST 至少包含主题切换。

#### Scenario: 查看用户管理
- **WHEN** chef 进入用户管理页面
- **THEN** 系统展示用户列表、头像、角色信息和可用操作

#### Scenario: 编辑用户信息
- **WHEN** chef 点击用户操作列的"编辑"按钮
- **THEN** 系统打开编辑抽屉，显示用户名输入框和头像上传区域（支持 1:1 裁剪）

#### Scenario: 修改用户名
- **WHEN** chef 修改用户名并提交，新用户名不存在
- **THEN** 系统更新用户名，刷新列表，展示成功反馈

#### Scenario: 用户名重复
- **WHEN** chef 修改用户名并提交，新用户名已存在
- **THEN** 系统提示"用户名已存在"，不关闭抽屉

#### Scenario: 上传头像
- **WHEN** chef 在编辑抽屉中选择图片并完成裁剪上传
- **THEN** 系统上传头像，更新用户头像，列表中显示新头像

#### Scenario: 新建用户
- **WHEN** chef 在用户管理页面提交合法用户名、初始密码和角色
- **THEN** 系统创建用户，刷新用户列表，并展示成功反馈

#### Scenario: 食客不能管理用户
- **WHEN** diner 访问用户或角色管理页面
- **THEN** 系统拒绝访问或隐藏管理入口

#### Scenario: 查看角色能力
- **WHEN** chef 进入角色管理页面
- **THEN** 系统从共享角色配置渲染 chef 与 diner 的名称、描述和权限说明
